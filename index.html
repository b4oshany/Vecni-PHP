Vecni is a light-weight PHP micro-framework that utilizes url dispatching to server files instead of filesystem url resolution. It utilizes Twig, which is equivalent to Python Jinja2 templates for advance template management.

Getting Started

Vecni uses a number of components such as Twig for templating, Less for css pre-processing amongst others that will be mentioned in the Tech Stack section. In the root folder of the application, run the following git commands (git bash command line):

$ git submodule update --init

This will perform a git clone on external modules that are needed..

Database Configuration

In the etc/configs folder there is a settings.ini.php file which contains basic server configuration. You can add your database settings or any settings you need to be rendered globally.

Apache Configuration

In order for the application to use url dispatching, one must enable the rewrite module in Apache.

Click here to enable rewrite module for Linux. Click here to enable rewrite module for Windows.

Enabling Apache rewrite module allows Vecni to be used at full capacity, where by all request to the Vecni base application is have to be validated via the index page. This will disallow direct POST and GET requests to files and folders.

Defining A URL

Setting a Basic Route

The main.ini.php file is where all your URL Routes will be defined. In order for you to defined a url, you must call the set_route function from the Vecni package. The set_route function takes in two parameters, the relative url to be rendered and the function name that will handle the rendered url as seen below.


Vecni::set_route("/", "home"){

function home(){return "Welcome to Vecni."; }

When a user visits the homepage of your site, which is indicated by "/" (this stands for the root url for site or application), Vecni will find and execute the home function. As a result, "Welcome to Vecni." will be displayed to the user.

Routes with variables.

If dynamic url is need for a path, then url variables can be passed into the set_route function by placing a variable between curl bracels in the dispatched url at it's respective path as seen below.


  Vecni::set_route("/book/{book_name}", "book_view")
  function book_view(){
    
      $book_name = Request::GET("book_name")
      return "You are viewing $book_name."
    }

Therefore, if a user visits http://localhost/vecni/book/Harry Potter/, the following will be shown:

You are viewing Harry Potter.
