Defining A URL

Setting a Basic Route

The main.ini.php file is where all your URL Routes will be defined. In order for you to defined a url, you must call the set_route function from the Vecni package. The set_route function takes in two parameters, the relative url to be rendered and the function name that will handle the rendered url as seen below.

Vecni::set_route("/", "home")
function home(){return "Welcome to Vecni."; }

When a user visits the homepage of your site, which is indicated by "/" (this stands for the root url for site or application), Vecni will find and execute the home function. As a result, "Welcome to Vecni." will be displayed to the user.

Routes with variables.

If dynamic url is need for a path, then url variables can be passed into the set_route function by placing a variable between curl bracels in the dispatched url at it's respective path as seen below.

Vecni::set_route("/book/{book_name}", "book_view") function book_view() $book_name = Request::GET("book_name") return "You are viewing $book_name."

Therefore, if a user visits http://localhost/vecni/book/Harry Potter/, the following will be shown:

    You are viewing Harry Potter.

Link Css

Move the plugin CSS file to your server and link to it in the head section of your document. 

      
    
      
  1. <link rel="stylesheet" href="yourplugin.css" />

Link JS

Move the plugin JS file to your server and link to it before the closing body tag.

Please note that you must include jQuery (v1.7 or later) before the slider JS file.

  1. <script src="jquery-1.7.1.min.js"></script>
  2. <script src="jquery.yourplugin.min.js"></script>
